# 기술 요구사항 정의서 (TRD)

## 1. 문서 정보

- 프로젝트명: RustMine (가칭)
- 목표: Minecraft(자바 에디션 기준)와 동등한 핵심 게임플레이를 Rust로 구현
- 범위: 클라이언트/서버/월드 저장/렌더링/게임 로직 포함한 단일 코드베이스(모듈 분리)
- 대상 플랫폼: Windows 10/11 (초기), 이후 Linux/macOS 확장 고려
- 기준 버전: Minecraft Java Edition 1.20.x 기능을 1차 기준으로 삼되, 단계적 구현

## 2. 상위 목표 및 성공 기준

- 동등성: 기본 생존 모드에서 블록 채굴/설치/제작/전투/이동/밤낮/몬스터 스폰이 동일하게 동작
- 성능: 1080p 기준 중간 옵션에서 60 FPS 이상 유지 (중급 GPU)
- 안정성: 장시간 플레이(2시간 이상) 시 메모리 누수 없음, 크래시율 0.1% 미만
- 멀티플레이: 8인 동시 접속에서 서버 틱 20 TPS 유지

## 3. 시스템 아키텍처 요구사항

### 3.1 구성

- 클라이언트
  - 렌더링, 입력, UI, 오디오
  - 네트워크, 월드 캐시
- 서버
  - 월드 시뮬레이션, 게임 룰, 엔티티 AI
  - 월드 저장/로드, 청크 관리
- 공통
  - 블록/아이템/레시피 데이터 정의
  - 물리/충돌/경로 탐색

### 3.2 모듈 분리

- `core`: 수학, 공통 유틸, 직렬화
- `world`: 청크, 블록, 바이옴, 구조물
- `entity`: 엔티티, AI, 물리
- `render`: 렌더링 파이프라인
- `net`: 프로토콜, 패킷, 동기화
- `audio`: 사운드 재생
- `ui`: HUD, 인벤토리, 메뉴
- `server`: 서버 런타임
- `client`: 클라이언트 런타임

## 4. 기능 요구사항

### 4.1 월드 및 청크

- 16x16x384(1.20 기준) 청크 구조 지원
- 바이옴 시스템 (온도/습도/색상 영향)
- 지형 생성: 오버월드(기본), 이후 네더/엔드 확장
- 구조물 생성: 마을, 던전, 폐광(단계적)

### 4.2 블록 및 아이템

- 기본 블록: 흙/잔디/돌/모래/자갈/나무/잎
- 광물: 석탄/철/금/다이아
- 도구: 나무/돌/철/다이아/네더라이트
- 블록 배치/파괴/드롭/수집 로직

### 4.3 제작 시스템

- 2x2 및 3x3 제작
- 기본 레시피 50개 이상 구현
- 제작대/화로(연료 소비 포함)

### 4.4 엔티티

- 플레이어
  - 이동, 점프, 달리기, 수영
  - 체력/허기 시스템
- 몬스터
  - 좀비/스켈레톤/크리퍼
- 동물
  - 소/돼지/닭

### 4.5 전투

- 근접 공격/원거리(활)
- 피격 판정 및 무적 시간
- 방어구/무기 내구도

### 4.6 네트워크

- 서버 권한 기반 동기화
- 플레이어 위치/블록 변경/엔티티 상태 실시간 동기화
- 핑 150ms 내에서도 정상 플레이 가능

### 4.7 UI

- HUD(체력/허기/경험치)
- 인벤토리/핫바
- 채팅창

### 4.8 오디오

- 기본 효과음(걷기, 채굴, 데미지)
- 배경음(오버월드)

## 5. 비기능 요구사항

### 5.1 성능

- 청크 로딩: 16 청크 이내에서 200ms 이하 로딩 목표
- 렌더 파이프라인: 60 FPS 이상 목표

### 5.2 안정성

- 충돌 방지 및 복구 가능한 월드 저장
- 저장 주기: 30초 기본

### 5.3 확장성

- 모드/플러그인 시스템 고려한 데이터 기반 설계

## 6. 데이터 및 포맷

- 월드 저장: Region 파일 포맷 또는 유사 구조
- 블록/아이템 데이터: JSON 기반

## 7. 테스트 요구사항

- 단위 테스트: 블록 배치/파괴/드롭
- 통합 테스트: 멀티플레이 동기화
- 성능 테스트: FPS/메모리 사용량 측정

## 8. 제약 및 가정

- 정식 상업 배포 전까지 사설 테스트 목적
- Minecraft 에셋 및 사운드는 사용 금지 (자체 제작 또는 오픈 라이선스 활용)
- 단계별 구현 (오버월드 -> 네더 -> 엔드)

## 9. 향후 확장 항목

- 레드스톤
- 포션/마법부여
- 엔드/네더 완성
- 커맨드/서버 관리 기능
